# 怎么写、怎么讲:金字塔方法

> **结论先行**:这本书的每一章,都按"先说结论,再给根据,再举例子"来写。你照这个方法填章节、写文档、做答辩,都能让对方一分钟听懂。

这一章既是**写作规范**,也是一项能直接用在课程答辩和工作里的技能。我们先讲为什么,再讲怎么做。

## 一、一分钟说清,否则三小时也不够

**结论**:如果一件事你不能在一分钟内说清楚,那讲三个小时也讲不清。

**根据**:听众的注意力和记忆有限——一场讲述,对方大约只记住其中 **10%**。把话讲长,不会让对方记得更多,只会让那 10% 落在无关紧要的地方。

**例子**:答辩时与其铺垫五分钟背景,不如开口就是"我们的用户中心解决了一件事:让多个系统共用一套登录"。这句话就是听众会记住的那 10%。

## 二、金字塔结构:结论 → 根据 → 例子

**结论**:任何一段表达,都organize成三层金字塔。

```
            ┌──────────┐
            │   结论    │   ← 你最想让对方记住的一句话
            └────┬─────┘
        ┌────────┼────────┐
     ┌──┴──┐  ┌──┴──┐  ┌──┴──┐
     │ 根据 │  │ 根据 │  │ 根据 │   ← 支撑结论的 2~3 个理由
     └──┬──┘  └──┬──┘  └──┬──┘
      ┌─┴─┐    ┌─┴─┐    ┌─┴─┐
      │实例│    │实例│    │实例│     ← 让根据可信、可感的具体例子
      └───┘    └───┘    └───┘
```

**根据**:先抛结论,对方立刻知道你要说什么,后面的内容他能"挂"在这个结论上;根据让结论站得住;例子让根据从抽象变具体。

**例子**:本章每一节都长这样——加粗的「结论」开头,「根据」解释,「例子」落地。

::: tip 写完必做:检查逻辑链
每搭完一个金字塔,从上往下读一遍:结论是不是真的由这几条根据推出来?根据之间有没有重叠或矛盾?例子是不是真的支撑那条根据?**逻辑不顺,宁可重搭,不要硬凑。**
:::

## 三、站在对方的位置上

**结论**:写之前,先把自己想象成坐在读者的位置,模拟"我这样说,他听得清吗"。

**根据**:讲述的目标从来不是"我说给你",而是"我们一起把一件事弄明白",最终让对方愿意为自己去做某件事。所以:

- 用「**我们**」而不是「我」——这是并肩,不是宣讲;
- **不搞纵向压制**:不是"我懂得多所以你听我的",也不是"他是组长所以我不敢提意见"。平等、尊重,该提意见就提;
- 让对方感到你在**帮他解决一件具体的事**,而不是在炫耀你知道多少。

**例子**:同样一句话,"你应该用 BCrypt"是命令;"**我们**用 BCrypt 来存密码,因为它能挡住库被脱后的暴力破解"是并肩解决问题。

## 四、图是补充,不是主角

**结论**:配图要精美但简单,用来补充而非替代你的话。

**根据**:这对应"快思考与慢思考"——图让人快速建立直觉,文字让人慢速理解细节。但图一旦太复杂,对方就只顾看图、不再听你讲,反而抢了主线。

**例子**:本书第二章那个用例图,只画了角色和几个用例,一眼能懂;真正的细节("账号≥4位")放在文字里讲。

## 五、走最短路径,别复述你的弯路

**结论**:直接把读者领到结果,不要描述"我是怎么一步步发现它的"。

**根据**:这本书不是科研论文。读者要的是"现在该怎么做",不是你的探索史。把省下来的篇幅,用来讲新技术**具体怎么用**,价值高得多。借用一段理工科写作的洞见:

> 论文是给读者读的,必须沿着最简明的路径来写,而不是自己当初发现时那条迂回曲折的路径。

**例子**:讲密码存储,我们直接说"用 BCrypt,这样存",而不是"我们先试了 MD5,发现不安全,又查了资料,最后换成 BCrypt"。后者是弯路,删掉。

## 附:什么是"思考"

**结论**:思考 = 用**知识**(你脑子里的内在信息)+ **消息**(外部进来的信息),得出**结论**。

**根据**:想清楚这一点,就知道金字塔的"结论"从哪来——它是知识和消息碰撞后的产物,不是凭空冒出的主张。

**例子**:看到"用户库被脱了密码全泄露"(消息),结合"哈希算法有快慢之分"(知识),得出"该用慢哈希 BCrypt"(结论)。这就是一次完整的思考,也正好是一个金字塔。

## 怎么用这一章

- 填任何占位章节前,先想:这一章的**一句话结论**是什么?
- 每一节用**加粗结论开头**,再写根据和例子;
- 写完**检查逻辑链**;
- 不确定要不要写某段?问自己:它是帮读者更快拿到结果,还是在复述弯路?后者删掉。
