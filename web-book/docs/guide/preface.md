# 序言:这本书是什么

## 缘起

我们在上《软件工程基础》。手头有三样东西:

1. **理论课件** —— 11 章 PDF,讲软件工程的方法论;
2. **项目笔记** —— 「用户中心」这个全栈项目的直播笔记(2022 年录的);
3. **文档模板** —— 课程要求提交的需求/设计/数据/测试等说明书。

问题是:这三样**各自独立**。理论是抽象的名词,笔记是具体的代码,文档是要交的格式——它们之间没有桥。而且笔记里的技术是 2022 年的,有些已经过时,甚至**不安全**(比如用 MD5 加盐存密码)。

## 这本书做了什么

这本书以「**用户中心**」一个项目为主线,把三样东西拧成一股绳:

- 讲**理论**时,旁边就是它在用户中心里的样子;
- 做**项目**时,用的是 **2026 的现代技术**,而不是过时的旧版;
- 每章都告诉你,对应的**课程文档**该怎么写。

> 一句话:**用一个项目,同时学会技术、学会软件工程、学会怎么交作业。**

## 为谁写

- **正在上软件工程课的同学** —— 理论 + 项目 + 文档一站式;
- **想做出第一个管理系统的人** —— 跟着实战篇从零做到上线;
- **我们团队自己** —— 这既是学习笔记,也是项目文档,还是给后来人的指南。

## 它和 2022 笔记的关系

我们**没有照抄**老笔记。老笔记是宝贵的起点,但这本书做了**现代化升级**:

| 维度 | 2022 笔记 | 本书(2026) |
| --- | --- | --- |
| 密码存储 | MD5 + 固定盐 ⚠️ | BCrypt(自带盐 + 慢哈希) |
| 登录态 | Session + Cookie | JWT / Spring Session + Redis |
| 前端构建 | Umi(重配置) | Vite(秒级热更新) |
| 部署 | 手动 Docker / 宝塔 | Docker Compose + CI/CD |
| 开发方式 | 纯手写 | AI 协作(Claude Code) |

完整对照见 [2026 技术全景图](/guide/tech-landscape)。
