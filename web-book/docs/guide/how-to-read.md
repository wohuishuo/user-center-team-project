# 如何阅读

这本书有三条主线,你不必从头读到尾。**先确认你是哪种人,再挑路线。**

## 三条阅读线

### 🎓 想先懂原理(应试 / 打基础)

```
理论篇(按章读) → 每章末尾跳到实战篇看真实代码 → 技术图鉴查细节
```

适合考试复习、想把概念吃透的人。理论篇每一章都标了"在用户中心里对应"哪一段实战。

### 🛠️ 想直接上手(做项目 / 学技术)

```
实战篇(从需求做到部署) → 卡住了去技术图鉴查那张卡 → 概念不懂回理论篇补
```

适合想尽快做出东西的人。实战篇是真实开发顺序,跟着做就有一个能跑的项目。

### 📄 要交作业(赶 deadline)

```
文档工坊(找到你要交的文档) → 跳转到书内对应章节抄思路 → 套用课程模板的格式
```

适合时间紧、先把作业交了的人。[文档工坊](/docs-workshop/) 有完整的"文档 → 章节"对照表。

## 阅读约定

- **★** 标记的章节是**已完整写好的样板**,质量最高,也最适合当模板;
- **`> TODO`** 表示这一段还是占位,等人填;
- **⚠️** 标记的是 2022 旧做法的"坑",现在不要再这么做;
- 右上角有 🔍 **搜索**,记不清在哪一章就直接搜关键词;
- 右上角可切换**深色 / 浅色模式**。
