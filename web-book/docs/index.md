## 这本书想解决什么

**结论:用一个项目,把理论、技术、文档三件事一次学会。**

我们的课程资料分成三堆:**理论课件**、**项目笔记**、**要交的文档模板**,三者各说各话,没有桥。这本书以「用户中心」一个项目为主线,左手讲清每个软件工程概念,右手用现在该用的技术把它实现出来,顺带告诉你对应的课程文档该怎么写。

> 读完这本书,你应该能独立做出一个管理系统,并且说得清「为什么这么做」。

整本书的写法也遵循同一套方法——**结论先行的金字塔结构**,见 [金字塔写作方法](/guide/writing-style)。

## 三条阅读线

| 你是谁 | 推荐路线 |
| --- | --- |
| 想先懂原理 | 理论篇 → 实战篇对应章节 → 技术图鉴查细节 |
| 想直接上手 | 实战篇从头做 → 卡住了去技术图鉴查 → 回理论篇补概念 |
| 要交作业 | 文档工坊 → 跳转到书内对应章节抄思路 |

不知道从哪开始?先看 [序言](/guide/preface) 和 [2026 技术全景图](/guide/tech-landscape)。
