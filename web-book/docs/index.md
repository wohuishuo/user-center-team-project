## 这本书想解决什么

我们的课程资料分成三堆:**理论课件**(PDF)、**项目笔记**(用户中心,2022 年录的)、**要交的文档模板**。三者各说各话,而且笔记里的技术已经过时——比如还在用 MD5 加盐存密码,这在 2026 是明确的安全隐患。

这本书把三者**拧成一股绳**:以「用户中心」一个项目为主线,左手讲清每个软件工程概念,右手用 2026 现代技术把它实现出来,顺带告诉你对应的课程文档该怎么写。

> 读完这本书,你应该能独立做出一个管理系统,并且说得清「为什么这么做」。

## 三条阅读线

| 你是谁 | 推荐路线 |
| --- | --- |
| 想先懂原理 | 理论篇 → 实战篇对应章节 → 技术图鉴查细节 |
| 想直接上手 | 实战篇从头做 → 卡住了去技术图鉴查 → 回理论篇补概念 |
| 要交作业 | 文档工坊 → 跳转到书内对应章节抄思路 |

不知道从哪开始?先看 [序言](/guide/preface) 和 [2026 技术全景图](/guide/tech-landscape)。
