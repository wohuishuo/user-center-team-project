# 软件工程理论篇

本篇对应《软件工程基础》课程的 **11 章理论**。和课件最大的不同是:这里每一个概念都**落到「用户中心」这一个项目上**,理论旁边就是它在我们项目里长什么样。

## 本篇章节

| 章 | 主题 | 在用户中心里对应 |
| --- | --- | --- |
| [第1章](/theory/01-overview) | 软件工程概述 | 为什么要按工程方法做用户中心 |
| [第2章 ★](/theory/02-requirements) | 软件需求工程 | 登录/用户管理/用户校验的需求是怎么来的 |
| [第3章](/theory/03-design) | 软件设计基础 | 分层架构、模块划分 |
| [第4章](/theory/04-structured-design) | 结构化设计方法 | 数据流图、模块结构图 |
| [第5章](/theory/05-implementation) | 软件实现 | 编码规范、复用与重构 |
| [第6章](/theory/06-testing) | 软件测试 | 单元测试 / 集成测试 |
| [第7章](/theory/07-uml) | UML 建模语言 | 用例图、类图、时序图 |
| [第8章](/theory/08-oo-analysis) | 面向对象分析 | 领域模型 |
| [第9章](/theory/09-oo-design) | 面向对象设计 | 设计模式、职责分配 |
| [第10章](/theory/10-maintenance) | 软件维护 | 迭代、缺陷修复 |
| [第11章](/theory/11-project-management) | 软件项目管理 | 分工、进度、风险 |

> ★ 标记的是**已完整写好的样板章节**,可作为填写其余章节的范例。

::: tip 怎么用这一篇
理论看不懂时,直接跳到「在用户中心里对应」那一栏指向的[实战篇](/project/)章节,看它的真实代码,再回来看概念,会快很多。
:::
