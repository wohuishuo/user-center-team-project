# 软件工程理论篇

> **结论先行**:这一篇把课程理论**精简成 6 章**,每个概念都落到「用户中心」项目上——理论旁边就是它在我们项目里的样子。

课程原本有 11 章,我们按"够用、不重复"的原则合并成 6 章:设计相关的(结构化、面向对象、UML)合到一章,维护和项目管理合到一章。**学方法,不背章节号。**

## 本篇章节

| 章 | 主题 | 在用户中心里对应 |
| --- | --- | --- |
| [1](/theory/01-overview) | 软件工程概述 | 为什么要按工程方法做用户中心 |
| [2](/theory/02-requirements) | 软件需求工程 | 登录/用户管理/用户校验的需求怎么来的 |
| [3](/theory/03-design) | 软件设计(结构化/面向对象/UML) | 分层架构、领域模型、用例图与类图 |
| [4](/theory/04-implementation) | 软件实现 | 编码规范、复用与重构 |
| [5](/theory/05-testing) | 软件测试 | 单元测试 / 集成测试 |
| [6](/theory/06-management) | 软件维护与项目管理 | 迭代、分工、进度、风险 |

> 六章均已写实;[第2章 需求工程](/theory/02-requirements) 结构最完整,可作新增章节参考。

::: tip 怎么用这一篇
理论看不懂时,直接跳到"在用户中心里对应"那一栏指向的[实战篇](/project/)章节,看真实代码,再回来看概念,会快很多。每章都按 [金字塔结构](/guide/writing-style)(结论→根据→例子)来写。
:::
