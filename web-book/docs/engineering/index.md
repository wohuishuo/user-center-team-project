# 软件工程与团队管理

技术之外,**怎么和人一起把项目做完**同样重要。本篇讲流程、协作和 2026 时代的 AI 协作方式。

## 本篇章节

| 章节 | 内容 |
| --- | --- |
| [软件生命周期](/engineering/lifecycle) | 需求→设计→开发→测试→部署→维护的全流程 |
| [Git 协作与分支策略](/engineering/git-workflow) | 主分支保护、feature 分支、PR 流程 |

## 企业开发流程速览

```
需求分析 → 设计 → 技术选型 → 初始化 → 写 Demo
   → 写业务代码 → 测试 → 代码评审 → 部署 → 发布上线
```

每一步在用户中心里都有对应实践,详见 [实战篇](/project/) 与 [软件生命周期](/engineering/lifecycle)。

::: tip 2026 的不同
最大的变化是 **AI 进入了每个环节**:需求可以让 AI 帮你梳理用例,骨架可以让 Claude Code 生成,测试可以 AI 起草,文档可以 AI 填初稿。人的角色从"逐行写"变成"设计 + 审校"。详见 [AI 协作工作流](/engineering/ai-workflow)。
:::
